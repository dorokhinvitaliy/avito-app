# Система модерации объявлений Авито

Веб-приложение для модерации объявлений на платформе Авито. Упрощенная версия внутренней системы модераторов.

## Функциональность

### Обязательные функции

- **Список объявлений** - просмотр, фильтрация, сортировка, пагинация
- **Детальный просмотр** - полная информация об объявлении, история модерации
- **Статистика** - метрики эффективности модератора, графики активности
- **Модерация** - одобрение, отклонение, возврат на доработку

### Дополнительные функции

- **Горячие клавиши** - быстрая навигация и принятие решений
- **Material Design** - современный UI/UX
- **Адаптивный дизайн** - работа на разных устройствах
- **Docker поддержка** - контейнеризация приложения

- **CORS** - используется для борьбы с CORS-ORIGIN в backend

## Технологии

### Обязательные

- **Node.js v20** - среда выполнения
- **React v18** - пользовательский интерфейс
- **TypeScript** - статическая типизация
- **React Router DOM** - навигация между страницами
- **Material-UI (MUI)** - библиотека компонентов

### Дополнительные

- **Axios** - HTTP-клиент для работы с API
- **Docker** - контейнеризация приложения
- **Vite** - сборка проекта

## Установка и запуск

### Запуск в Docker

```bash
git clone https://github.com/dorokhinvitaliy/avito-app.git
cd avito-app

docker compose up --build
```

Фронтенд: http://localhost:5173

Бэкенд: http://localhost:3001

### Локальный запуск

1. **Клонирование репозитория**

```bash
git clone https://github.com/dorokhinvitaliy/avito-app.git
cd avito-app
```

2. **Установка зависимостей**

```bash
cd client
npm install
```

3. **Запуск бэкенда**

```bash
cd server
npm install
npm start
# Бэкенд доступен на http://localhost:3001
```

4. **Запуск фронтенда**

```bash
npm run dev
# Фронтенд доступен на http://localhost:5173
```

### Продакшн сборка

```bash
# Сборка фронтенда
npm run build

# Просмотр продакшн сборки
npm run preview
```

## Обоснование выбора технологий

### TypeScript

JavaScript имеет динамическую типизацию, что приводит к ошибкам во время выполнения в больших приложениях.

TypeScript предоставляет статическую типизацию, которая обнаруживает ошибки на этапе компиляции, улучшает читаемость кода и автодополнение и облегчает рефакторинг и поддержку

### Material-UI (MUI)

Создание UI компонентов с нуля требует значительных временных затрат.

MaterialUI предлагает готовые компоненты, а так же встроенную темизацию и кастомизацию.

### Vite

Create React App имеет медленную сборку в больших проектах.

Vite обеспечивает мгновенный запуск dev сервера, быструю горячую перезагрузку, а так же оптимизированную продакшн сборку.

### Axios

Нативный fetch API имеет ограниченную функциональность.

Axios предоставляет автоматические преобразования JSON, межсайтовые куки по умолчанию, таймауты запросов, интерцепторы для логирования и обработки ошибок.

### Docker

Различия в окружениях между разработкой и продакшеном.

Docker решает многие проблемы и делает процесс запуска и сборки удобным.
